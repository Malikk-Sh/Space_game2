# SINTARA — Roadmap к "шедевру"

> Этот документ — план развития игры по итогам глубокого аудита (геймплей, визуал, аудио, нарратив, UX). Редактируйте свободно — расставляйте `[x]`, меняйте приоритеты, добавляйте свои идеи.

## Контекст

**Текущее состояние**: ~7.5/10. Технически крепкая инди-игра с проработанным боссом и атмосферным Дрошем. Но эмоционально инертна, бой монотонен, корабль и пара планет почти не используются.

**Цель**: 8.5–9/10. Игра, которую запоминают и хочется советовать.

**Ключевые проблемы, которые надо решить**:
1. **Монотонный космос** — один тип врагов (пираты), два оружия, линейная сложность
2. **Слепая навигация** — маяки и NPC без маркеров, игрок 5+ минут ходит наугад
3. **Безликий нарратив** — пришелец без имени, NPC одноразовы, финал — голый текст
4. **Корабль = меню** — 4 комнаты, 2 действия (заправка, постройка)
5. **Скачок сложности в фазе 3 Тины** — отскоки пуль убивают, нет градиента
6. **Бубблика и Краснозём визуально неотличимы** — почти стандартный космический фон
7. **Прогресс теряется при F5** — нет localStorage

## ФАЗА 2 — Глубина геймплея (60% выигрыша)

### 2.1 Новые типы врагов

Сейчас в космосе один враг — пират (см. `12-scene-space.js`, `drwPirate`). Боевая система одномерна. Добавляем 3 типа + мини-босс:

#### A. Танк (`tank`)
**Роль**: медленный, тяжёлый, заставляет двигаться и расходовать энергию.
- HP: **80** (vs 20 у пирата)
- Скорость: **0.4** (vs 0.7)
- Атака: медленные крупные снаряды каждые **60–90 кадров**, урон **8**, размер пули x2
- Визуал: угловатый широкий корпус, тёмно-красный (`P.PIR2`), пульсирующее ядро
- Спавн: `G.prog > 0.4`, реже пиратов (вес 0.3)
- Дроп: гарантированно **2 RES + 30 CR**

#### B. Дрон-рой (`drone`)
**Роль**: быстрый рой, заставляет уворачиваться. Камикадзе.
- HP: **4** (фарф)
- Скорость: **1.5** (быстрее игрока)
- Атака: НЕ стреляет, таран за **5 урона** при контакте
- Визуал: маленький синий треугольник (`P.L1`) со светящимся хвостом из частиц
- Спавн: группами **3–5 штук**, `G.prog > 0.2`
- Дроп: **1 RES + 5 CR** за штуку

#### C. Снайпер (`sniper`)
**Роль**: дальняя угроза с телеграфом. Прерывает фарм.
- HP: **25**
- Скорость: **0.1** (почти стоит)
- Атака: заряжает выстрел **60 кадров** (видимый красный луч-телеграф) → мгновенный высокоурон снаряд (**15 dmg**)
- Визуал: тонкий вытянутый корпус, оптический прицел светится красным
- Спавн: появляется **1 раз за полёт**, `G.prog > 0.3`
- Дроп: **2 RES + 15 CR + шанс на энергию**

#### D. Мини-босс (`miniboss`)
**Роль**: пик каждого полёта, кульминация боя.
- HP: **150**, 2 фазы (стрельба → таран при HP < 50%)
- Скорость: **0.5**, в фазе тарана разгоняется до 1.8
- Визуал: увеличенный пиратский корабль x2, шрамы, флаг
- Спавн: один раз в каждом полёте, после `G.prog > 0.7`
- Дроп: гарантированно **80 CR + 1 материал апгрейда** (см. ниже)

**Изменения в коде**:
- `G.enms[]` элементы получают поле `type: 'pirate' | 'tank' | 'drone' | 'sniper' | 'miniboss'`
- `updEnemyByType(e)` диспетчер в `updSpace` — отдельная AI-логика на тип
- `drwEnemyByType(e)` в `09-entities.js` — отдельный рендер
- В спавн-логике (`12-scene-space.js` ~ строка 156) — взвешенный выбор типа по `G.prog`

### 2.2 Расширенный арсенал оружия

Сейчас: L1 (2 dmg, 10 EN) и L2 (10 dmg, 44 EN). Линейная прогрессия → нет смысла менять. Делаем **дерево из 6 опций**:

| Оружие | Урон | EN | Темп стрельбы | Особенность |
|--------|------|----|----|----|
| **L1 (default)** | 2 | 10 | быстрый | сейчас существует |
| **Spread** | 1×3 | 18 | средний | веер из 3 пуль 30° |
| **Missile** | 5 | 25 | медленный | самонаведение на ближайшего |
| **L2** | 10 | 44 | средний | существует, через квест Дроша |
| **Beam** | 0.5/frame | 1/frame | continuous | держать кнопку, расход энергии безостановочно |
| **Burst** | 3×5 | 50 | очень медленный | очередь из 5 пуль за 30 кадров |

Получение:
- L1 — стартовое
- Spread — крафт на корабле, 50 CR + 1 материал
- Missile — за побочный квест Краснозёма
- L2 — текущий квест Дроша (без изменений)
- Beam — крафт на корабле, 120 CR + 2 материала
- Burst — найти чертёж на Бубблике (новая зона)

Управление: цифровые клавиши **1–6** для переключения; на тач — циклический свич-баттон в HUD.

**Изменения в коде**:
- `G.pl.wep` — теперь массив `[1, 'spread', null, ...]` (доступные слоты)
- `G.pl.wepCurrent` — индекс текущего
- В `updSpace` — расширить блок стрельбы (~строка 143) на switch по типу
- В HUD — заменить две иконки L1/L2 на подпись типа оружия

### 2.3 Ребаланс финала Тины

Сейчас 4 фазы с резким скачком сложности в фазе 3 (отскоки пуль убивают, требует синхронизации с орбитой брешей). Делаем **7 фаз с плавным градиентом** и контрольными точками:

| Фаза | Триггер | Механика | Сложность |
|------|---------|----------|-----------|
| 1 | старт | 3 энергоблока + щит + 3 паттерна атаки | ★☆☆☆☆ |
| 1.5 | 2/3 блока уничтожены | Скорость x1.2, новый паттерн (спираль) | ★★☆☆☆ |
| 2 | 0 блоков, щит пал | Турели + прямой урон по корпусу | ★★☆☆☆ |
| 2.5 | турели на 50% HP | 2–3 дрона | ★★★☆☆ |
| 3 | Тина HP < 50% | **ОДНА брешь** в щите, медленная (0.005 rad/frame) | ★★★★☆ |
| 3.5 | Тина HP < 30% | **ДВЕ бреши**, скорость 0.010 | ★★★★☆ |
| 4 | Тина HP < 15% | 3 бреши + rage-режим + частые дроны | ★★★★★ |
| Финал | HP < 5% | Текущая катсцена эмерженси-протокола (без изменений) | (story) |

**Контрольные точки**: `saveCheckpoint(G, 'finale_phase_X')` при переходах **1→2, 2→3, 3→4**. При смерти откатывает не к началу боя, а к началу текущей фазы.

**Изменения в коде**:

- В `02-checkpoint.js` — расширить `saveCheckpoint` для finale-чекпоинтов
- При смерти в финале — функция `restoreFinalePhase(G)` восстанавливает Тину на нужном HP

**Окрашенные снаряды**:
- В фазах 3–4 пули, которые **отскочат** (попадут не в брешь), визуально другого цвета (`P.YEL` жёлтый с красной обводкой)
- Игрок видит "куда стрелять можно, куда нет" с первого кадра

### 2.4 Корабль как мини-игра

Сейчас 4 комнаты, реальных действия два — заправка и постройка. Делаем **систему распределения рабочих** и **очередь крафта**:

#### Распределение рабочих
Каждая комната может иметь **0–3 рабочих**. Игрок назначает (тап на комнату → слайдер). Бонусы:

| Комната | Эффект на 1 рабочего |
|---------|-----|
| **Power** | +0.18 регенерации энергии в космосе (текущий механизм) |
| **Workshop** | +1 frame/кадр прогресса крафта |
| **Bridge** | +5% снижение урона по кораблю в полёте |
| **Fuel** | +5% эффективности использования топлива |

Стартово: 2 рабочих, оба в Power. Зорп добавляет 3-го. Скрытый бонус Краснозёма даёт 4-го.

#### Очередь крафта
Сейчас крафт мгновенный. Делаем **очередь**:
- Игрок ставит чертёж в очередь (CR списываются сразу)
- В полёте каждые 60 кадров очередь продвигается на `1 * workersInWorkshop` единиц
- При завершении — `G.notif` + `sfxAchievement` + предмет в инвентаре

Это даёт смысл балансу: больше рабочих в Workshop = быстрее апгрейды, но меньше энергии в бою.

**Изменения в коде**:
- `G.ship.workers = {power: 2, workshop: 0, bridge: 0, fuel: 0}`
- `G.ship.craftQueue = [{item: 'spread', progress: 0, total: 600}, ...]`
- В `updSpace` — продвигать `craftQueue` каждые 60 кадров
- В `15-scene-ship.js` — UI для слайдера рабочих + список очереди

## ФАЗА 4 — Атмосфера планет

Дрош уже отличный (метель, иней, ледяной пол, дыхание игрока). Бубблика и Краснозём почти без визуала. Делаем им оригинальность.

### 4.1 Бубблика (газовая планета)

**Палитра**: уже есть `P.BUB1=#cc88cc, BUB2=#8844aa, BUB3=#ffaaee, BUB_ATM=#331144, BUB_GAS1=#5a2266, BUB_GAS2=#7a3388, BUB_GAS3=#9944aa`.

**Что добавить**:

#### Фон
- Вертикальный градиент: `BUB_GAS1` (тёмный сверху) → `BUB_GAS2` (середина) → `BUB_GAS3` (низ)
- Реализация: 60 горизонтальных полосок с интерполяцией цвета

#### Облачные слои

#### Пузыри
- 12–15 полупрозрачных кругов диаметром 8–30px
- Движение: vy = -0.2 до -0.4 (всплывают), плюс sin-колебание по X
- Реализация: массив `G.bubbles[]` в `initPlanetBubblika`, обновление в `updPlanetBubblika`, рендер в `drwPlanetBubblika`

#### Атмосферный шиммер

## ФАЗА 5 — Долгосрочность и replayability

### 5.1 Сохранения через localStorage

Сейчас `G.checkpoint` живёт в памяти, F5 = всё потеряно. Делаем **новый файл `js/19-persistence.js`**:

```js
const SAVE_KEY = 'sintara_save_v1';

function persistGame(G) {
  const data = {
    version: 1,
    timestamp: Date.now(),
    campaignState: G.campaignState,
    checkpoint: G.checkpoint,
    achievements: G.achievements || {},
    settings: { sfxOn: SFX_ON, musicOn: MUSIC_ON },
    playTime: G.playTime || 0,
  };
  try { localStorage.setItem(SAVE_KEY, JSON.stringify(data)); } catch(e) {}
}

function loadGame() {
  try {
    const raw = localStorage.getItem(SAVE_KEY);
    if (!raw) return null;
    const data = JSON.parse(raw);
    if (data.version !== 1) return null;  // несовместимая версия
    return data;
  } catch(e) { return null; }
}

function clearGame() {
  try { localStorage.removeItem(SAVE_KEY); } catch(e) {}
}
```

**Когда вызывать**:
- `persistGame(G)` — внутри `saveCheckpoint`, при покупке на верстаке, при завершении квеста, при выходе с планеты
- `loadGame()` — при инициализации, в главном меню
- `clearGame()` — при "Начать заново"

**Главное меню**:
- Если есть сохранение — показать кнопку "ПРОДОЛЖИТЬ (планета X, время Y:ZZ)"
- Кнопка "НОВАЯ ИГРА" с подтверждением "Уверены? Текущий прогресс будет удалён."
